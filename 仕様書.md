6次元将棋 仕様書 v0.1
1. ゲームの核となる定義
6次元将棋は、通常の盤面座標に加えて、以下の軸を持つ将棋である。
6つの軸
x軸: 筋
y軸: 段
t軸: 時間
b軸: 分岐世界線
l軸: 手番層
p軸: 位相
ただし、ここで重要なのは、6軸を全部同列の空間軸として扱わないことです。
実際の意味はこうです。
x, y は通常の盤面座標
t はその世界線内の手数
b は時間移動によって発生する別世界線ID
l は先手手番盤 / 後手手番盤の区別
p は位相（0または1）
このうち、プレイヤーが主に認識するのは、
盤面
時間
分岐
位相
です。
l は内部処理に近く、UI上ではそこまで強く見せなくてよいです。
2. コアコンセプト
このゲームの本質は次の一文で表せます。
「時間と分岐を渡る将棋に、位相世界を追加した“見えているのに届かない”将棋」
つまり、
過去へ干渉できる
その結果、新しい世界線が生まれる
しかも各世界線には位相0/1があり、駒の干渉可能範囲が変わる
これが6次元将棋です。
3. MVPの前提
最初から本将棋サイズでやると、ほぼ確実に実装が崩れます。
したがって v0.1 では MVPルールを採用します。
MVP盤面
盤サイズ: 5×5
MVP駒種
各陣営の初期配置駒:
王 1
飛 1
角 1
金 1
歩 2
合計 6駒
成り
歩 → 金 のみ
飛・角の成りは v0.2 以降で追加
金・王は成らない
持ち駒
あり
ただし打ち歩詰め禁止などの細かい将棋ルールは v0.1 では一旦無効でもよい
二歩だけは導入推奨
この時点で、普通の将棋としてはかなり軽量ですが、
時間・分岐・位相が入るので、十分重いです。

5. 駒の定義

type Piece = {

これがこのゲームの重要な特徴です。
6. 位相ルール
6次元目の中核です。
6.1 位相の基本
各局面には2つの位相層がある。
phase 0
phase 1
同じ (x, y) に phase 0 と phase 1 の駒が同時に存在してよい。
つまり、位相が違えば重なれる。
6.2 干渉原則
駒の移動・捕獲・利きは、原則として同位相内のみ有効。
例:
phase 0 の飛車は、phase 0 の駒にしか利かない
phase 1 の王は、phase 0 の角からは王手されない
6.3 位相移動
一部の駒は位相を変えられる。
v0.1 では次のようにする。
位相移動可能駒
王: 1マス移動の代わりに位相反転可能
角: 通常移動時に位相反転を伴ってもよい
歩: 位相移動不可
金: 位相移動不可
飛: 位相移動不可
これはかなり重要です。
全駒に位相移動を許すと一気に壊れます。
6.4 位相反転移動の扱い
例えば角が phase 0 から phase 1 に移る場合:
移動元: phase 0
移動先: phase 1
捕獲するなら、移動先位相にいる敵駒のみ捕獲可能
つまり、「位相をまたぐ移動」は、着地先位相を基準に干渉します。
7. 通常移動ルール
MVP用の駒の動きは以下。
7.1 王
8方向に1マス
追加で「その場位相反転」が可能
例: (x, y, phase0) → (x, y, phase1)
7.2 飛
縦横に任意マス
同位相のみ
位相反転不可
7.3 角
斜めに任意マス
通常斜行に加え、位相反転付き斜行も可能
つまり、
同位相の斜行
別位相への斜行 の両方ができる
これはこのゲームの花形です。
7.4 金
通常将棋の金と同じ
位相反転不可
7.5 歩
1マス前進
位相反転不可
成ると金扱い
8. 捕獲と持ち駒
8.1 捕獲
駒を取った場合、その駒は成りを解除して自分の持ち駒になる。
8.2 位相情報
捕獲した時点で、その駒の位相情報は失われる。
つまり持ち駒は常に位相未確定。
8.3 打ち駒
持ち駒を打つときは、以下を指定する。
x座標
y座標
phase 0 または phase 1
つまり、同じマスでも違う位相なら打てる。
8.4 二歩
同一世界線・同一位相内で、同一筋に自分の未成歩が2枚存在する打ちは禁止。
この「同一位相内」という条件がポイントです。
phase 0 に歩がいても、phase 1 には歩を打てる。
9. 時間移動ルール
ここが5D系の心臓部です。
9.1 時間移動の原則
一部の手は、現在の局面ではなく、過去の局面を対象にして行える。
ただし、過去局面に直接上書きはしません。
代わりに、そこから新しい世界線を生成します。
9.2 誰が時間移動できるか
v0.1 では簡潔にするため、以下の2種類のみ許可。
王を除く駒の通常移動を、過去局面に対して行える
打ち駒を、過去局面に対して行える
ただしこれだと強すぎるので、制限を入れます。
9.3 時間移動可能範囲
現在の局面から見て、最大2手前までの過去局面にのみ干渉可能。
例: 現在が turn = 10 なら、
turn = 9
turn = 8 への干渉のみ可能
これはMVPでは必須です。
無制限にすると探索木が壊れます。
9.4 時間移動手の結果
過去局面に対して手を行った場合:
その過去局面を親として
新しい timeline を生成し
以後その新世界線でゲームが進む
元の世界線は残る。
10. 分岐世界線ルール
10.1 世界線生成
新分岐は以下の情報を持つ。
Ts
コードをコピーする
type TimelineMeta = {
  id: number;
  parentId: number | null;
  branchedFromTurn: number | null;
  createdByPlayer: Player | null;
};
10.2 世界線の継承
分岐元の過去局面の状態を完全コピーし、その手を適用したものが新世界線の初期局面となる。
10.3 世界線の独立性
一度分岐した世界線は、その後は独立して進行する。
他世界線にある駒は、その世界線にしか存在しない。
11. 世界線安定度
これが6次元将棋を単なるカオスで終わらせないための装置です。
11.1 安定度の定義
各世界線には stability を持たせる。
初期値:
初期世界線: 5
分岐して生まれた新世界線: 親世界線の安定度 - 1
11.2 安定度コスト
以下の行為で安定度を消費する。
過去局面への干渉: -1
さらにその分岐世界線から再度過去干渉: -1
同一プレイヤーが連続して分岐生成した場合、追加で -1
11.3 崩壊
安定度が 0 になった世界線は崩壊候補となる。
v0.1 では処理を簡略化して、
自分の手番開始時に stability <= 0 の世界線では行動できず敗北扱い
とするか、あるいは
その世界線は以後選択不能
とする。
後者の方が扱いやすいです。
11.4 理由
安定度がないと、理論上は分岐を量産するのが最適戦略になりやすい。
それを抑えるための資源です。
12. 手番進行
ここは非常に重要です。
5D系で最も分かりにくくなりやすい部分です。
v0.1 では、各ターンでプレイヤーは1つの世界線・1つの局面に対して1手だけ行うものとします。
12.1 ターンの流れ
手番プレイヤーが、操作対象の世界線を1つ選ぶ
その世界線の最新局面に対して
通常手
打ち駒
過去干渉による分岐生成 のいずれかを行う
その結果生じた局面が新しい最新局面になる
手番交代
12.2 最新局面の定義
各世界線には「最新手数」がある。
通常はそこに対してのみ通常手を行える。
12.3 過去干渉
過去局面を選んで分岐生成した場合も、それがその手番の1手になる。
13. 王手と合法手
13.1 王手
ある世界線・ある局面・ある位相に存在する自玉が、敵駒の利きに晒されている状態。
13.2 王手の判定範囲
王手判定は同一世界線・同一局面の中でのみ行う。
他世界線から直接王手はかからない。
これは重要です。
他世界線まで王手判定に含めると、ゲームがほぼ人類に読めなくなります。
13.3 合法手条件
手を指した結果、
自分の王が存在する全位相について
その世界線の新局面で王手されていない
必要がある。
13.4 王の複数性
各世界線には、それぞれ王が1つずつ存在し得る。
分岐によって王もコピーされるためです。
これは仕様上かなり特殊ですが、5D系では避けられません。
14. 勝利条件
v0.1 では複雑にしすぎないため、勝利条件は次とします。
14.1 基本勝利
相手がどの選択可能世界線においても合法手を持たない場合、自分の勝ち。
14.2 代替簡略ルール
MVPではさらに簡略化して、
相手の“現在選択中世界線”の王を詰ませたら勝ち
でもよいです。
ただ、これは6次元感が弱くなるので、本命は前者です。
14.3 崩壊勝ち
相手の残存世界線が全て安定度崩壊で選択不能になった場合も勝ち。
15. 千日手・ループ対策
高次元将棋ではここを甘く見ると終わります。
15.1 同一局面判定
以下が一致したら同一局面とみなす。
世界線ID
手番
phase0盤面
phase1盤面
両者持ち駒
安定度
15.2 反復
同一世界線で同一局面が3回出たら、その世界線は引き分け候補。
15.3 推奨MVP処理
MVPでは
同一世界線で3回反復したら、その手は禁止
とするのが実装上は楽です。

  type: "drop";
  timelineId: number;

20. AI設計方針

王手回避を優先
分岐生成を乱発しない
20.2 評価関数
初期版はこれで十分です。
王の安全度
駒得
持ち駒価値

分岐数
安定度残量
例:
Ts

  materialScore
  + kingSafetyScore

20.3 重要な補正
分岐手は面白いですが、AIは雑に乱発しがちです。
そのため、
安定度消費が重い分岐は減点
即王手になる分岐は加点

が必要です。

24. 一番大事な設計判断
このゲームを作る上で最重要なのは次です。

その方向はほぼ確実に失敗します。
成功しやすいのは、
6次元目に意味を持たせる
その意味をプレイヤーが読める形にする
UIで追える範囲に制限する
分岐に資源コストを課す
